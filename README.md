我对于金融市场的波动很感兴趣，计划在传播学等领域建立指标，分析社会思潮的变化

指标设计
- 粉丝量：自媒体粉丝量，计算蓝筹作者、小作者之类的作者指数
	- 播放量：以视频为核心，计算某类 Tag 的指数（后续考虑）
- 数据来源：抖音、哔哩哔哩等交易所数据，打 Tag、排序
- 指数设计：$\sum\limits_{}^{}Followers/Divisor$
- 成分股设计：每年调整一次成分股，比如蓝筹100

指标分析
- 牛市背离：蓝筹很稳、新兴指数暴涨，说明思潮变化加速
- 板块轮动：知识类下跌、玄学类上涨，代表社会开始从外求内

## 设计

目录
- data：`.db` 文件，包含平台、作者、粉丝量、Tag、时间等信息
- src：源代码
- frontend：纯 HTML，方便数据可视化并部署到 Github Pages
- temp：临时文件、参考代码等

## 实施方案清单

### 数据库设计
- [ ] 使用 SQLite 单机版（零运维，足够支撑百万级记录）
- [ ] 多标签设计：author_tags 中间表关联，一个 UP 主可打多个 Tag
- [ ] 粉丝历史表：按时间序列存储（author_id + recorded_at 联合主键）

### UP 列表与 Tag 管理
- [ ] config.json 作为唯一数据源（UP主 UID、名称、标签数组）
- [ ] 本地单文件 HTML，浏览器直接打开使用
- [ ] UP 主管理：导入/导出 config.json、添加/删除/编辑、多标签选择、搜索过滤
- [ ] 指数设计器：可视化筛选条件

### 算法与指数设计
- [ ] 算法：粉丝量求和，除以平滑项
- [ ] 成分股筛选：蓝筹指数设置门槛（如粉丝量前100）、Tag 筛选规则
- [ ] 再平衡策略：定期调整成分股时的除数修正算法

### 自动刷新方案（GitHub Actions）
- [ ] Actions workflow：定时触发（cron）
- [ ] 执行流程：检出代码 → 安装依赖 → 执行爬虫 → 生成 frontend HTML → commit 回仓库
- [ ] GitHub Pages 自动部署：监测到 frontend/ 更新即重新部署
- [ ] 反爬应对：降低请求频率、如需登录则配置 Secrets、测试 Actions IP 可用性

### 前端页面设计（frontend/）
- [ ] 指数大盘：展示各 Tag 指数实时值、涨跌幅、走势图（日线/周线/月线）
- [ ] 板块对比：多指数叠加对比，支持牛市背离、板块轮动分析
- [ ] 成分股详情：点击指数查看成分列表，显示各 UP 主贡献度
- [ ] 历史回测：支持选择日期区间，回放指数历史走势
- [ ] 移动端适配：图表自适应，核心数据优先展示
